# Orbit v0.3.0 — Storage & RAG (GCP) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GCS RAG-documents bucket, an Artifact Registry Docker repo, and pgvector-on-Cloud-SQL wiring to the Astromesh Orbit GCP provider, shipping it as v0.3.0.

**Architecture:** Orbit renders Terraform HCL from Jinja2 templates via a per-provider plugin (`GCPProvider`). New storage resources are opt-in Jinja templates driven by a new `spec.storage` config section; the runtime service account is IAM-wired to them, and the runtime container gets `ASTROMESH_RAG_BUCKET`. The RAG vector store reuses the already-provisioned Cloud SQL (pgvector) — no new managed DB — so that part is config/docs only.

**Tech Stack:** Python 3.12+, Pydantic v2, Jinja2, Terraform (GCP: `google_storage_bucket`, `google_artifact_registry_repository`), pytest (`asyncio_mode=auto`), uv.

## Global Constraints

- Package under work: `astromesh-orbit/` (standalone; no dependency on core `astromesh`).
- Run tests with: `uv run --extra dev --with pytest pytest -q` from `astromesh-orbit/`.
- Config `apiVersion` must be `astromesh/v1`, `kind` must be `OrbitDeployment`.
- `OrbitMetadata.environment` is `Literal["develop", "staging", "production"]`, default `"develop"` (changed in `82cb760`; tests drifted — see Task 1).
- All new Terraform resources render ONLY when their `enabled` flag is true (no dangling refs when disabled).
- GCS bucket name pattern: `${project_id}-<metadata.name>-rag-docs` (project IDs are globally unique).
- Artifact Registry default `repository_id`: `<metadata.name>-images` when `repository` is empty.
- Runtime SA resource is `google_service_account.orbit` (from `iam.tf.j2`); its email is `${google_service_account.orbit.email}`.
- No live GCP calls in tests — render templates / build config only (matches the existing suite).
- Ruff: line length 100, target py312.
- Version lives in TWO files, bump both: `astromesh-orbit/pyproject.toml:3` and `astromesh-orbit/astromesh_orbit/__init__.py:3`.
